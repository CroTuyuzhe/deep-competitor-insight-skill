# 竞品深度分析

## 使用流程

当用户触发竞品分析时，按以下三步完成：

### Step 1: 理解用户意图

从用户查询中提取：
- **行业**: AI陪伴/短视频/电商/SaaS 等
- **深度**: basic(概览) / standard(标准) / deep(深度) / strategic(战略级)
- **聚焦维度**: 会员设计/商业模式/增长策略/定价策略 等（可选）
- **市场区域**: cn(国内) / global(海外) / both(国内外)

### Step 2: 运行分析脚本

```bash
cd {baseDir}/scripts && pip3 install -q ddgs langgraph 2>/dev/null; python3 competitor_analysis.py \
  --query "<用户原始查询>" \
  --depth <basic|standard|deep|strategic> \
  --focus "<聚焦维度，如有>" \
  --region <cn|global|both> \
  --format json \
  --verbose
```

**CLI 参数说明：**

| 参数 | 缩写 | 说明 | 默认值 |
|------|------|------|--------|
| `--query` | `-q` | 用户查询（必填） | 必填 |
| `--industry` | `-i` | 指定行业（覆盖自动解析） | 自动解析 |
| `--depth` | `-d` | basic/standard/deep/strategic | standard |
| `--focus` | | 聚焦维度 | 无 |
| `--region` | | cn/global/both | both |
| `--format` | `-f` | json/text | json |
| `--verbose` | `-v` | 显示节点执行详情 | 关闭 |
| `--max-products` | | 最大产品数 | 8 |

**深度选择参考：**
- 用户说"简单看看"/"概览" → basic
- 用户说"分析一下"/"做个对比" → standard
- 用户说"深度分析"/"详细调研"/"全链路" → deep
- 用户说"战略级"/"给老板汇报"/"全面战略" → strategic

### Step 3: 基于 JSON 输出生成专业报告

脚本输出结构化 JSON 数据。你需要基于这些数据，运用你的分析推理能力，生成以下 9 章节的专业报告。

**核心原则：数据来自脚本，洞察来自你的推理。不要简单复述搜索结果，要分析、提炼、判断。**

#### 报告结构

**一、行业全景**
- 基于 `landscape` 数据，综合判断市场规模、增速、发展阶段
- 描述赛道分层（哪些细分、各自规模和特征）
- 若数据不足，基于你对行业的理解补充，并标明哪些是推断

**二、竞品矩阵总览**
- 基于 `products` 列表，构建产品定位图
- 按维度分类：头部玩家 / 新兴挑战者 / 垂直玩家 / 海外对标
- 一句话概括每个产品的核心定位

**三、产品深度拆解**
- 基于 `product_profiles` 数据，对每个产品进行深度分析
- 不只列功能清单，要分析：为什么这样设计？背后的产品策略是什么？
- 指出每个产品做得好和做得差的真正原因

**四、聚焦维度分析**（若用户指定了聚焦维度）
- 基于 `dimension_analysis` 数据
- 跨产品横向对比，明确指出哪家做得最好、为什么
- 提炼最佳实践和常见陷阱

**五、用户痛点与机会**
- 基于 `pain_points` 数据
- 归类痛点：功能缺陷 / 体验问题 / 定价争议 / 内容质量 / 其他
- 将痛点映射为产品机会，按优先级排序

**六、多维度对比表**
- 基于 `comparison` 数据，生成格式化对比表格
- 维度：目标用户、核心功能、商业模式、定价、技术路线、内容生态、用户体量、差异化
- 在表格之外，分析竞争聚集点（大家都在做的）和空白地带（没人做好的）

**七、核心洞察**（最重要的章节）
- 基于所有数据 + 你的推理，提炼 3-5 条非显而易见的战略洞察
- 每条洞察必须包含：
  - 洞察本身（一句话）
  - 支撑证据（来自数据的具体依据）
  - 推理链条（为什么得出这个结论）
- 洞察类型参考：竞争格局规律、头部产品共性、本质差异、隐形壁垒、商业化有效/无效模式

**八、趋势预测**（deep/strategic 深度时）
- 基于 `trends` 数据 + 你的判断
- 功能演变趋势、用户偏好变化、商业化趋势、行业集中度趋势
- 每个趋势附置信度（高/中/低）和时间窗口
- 未来 6-12 个月可能发生的格局变化

**九、战略建议**
- 综合以上所有分析
- 提出 3-5 条可执行的战略建议
- 每条建议关联具体证据和数据
- 区分"必须做"/"值得做"/"可以观望"

#### 报告质量要求

- 不写空话套话，每一句都要有信息量
- 若某个维度数据不足，诚实说明而非编造
- 数字引用必须标注来源（来自搜索结果 or 推测）
- 洞察必须有推理链条，不能只有结论没有论证
- 报告应可直接用于商业决策或老板汇报

## 工作流

```
START
  │
  ▼

  │
  ▼
[行业全景] 搜索市场规模/格局/赛道分层
  │
  ▼
[产品筛选] 识别头部/代表性/垂直/海外产品
  │
  ▼
[产品深挖] 每个产品的定位/功能/商业模式/差异
  │
  ├─ 有聚焦维度 ──► [维度深度分析] 针对指定维度跨产品深度对比
  │                    │
  └─ 无聚焦维度 ──┐    │
                  ▼    ▼
              [用户痛点挖掘] 真实用户吐槽/不满/潜在需求
                  │
                  ▼
              [竞品对比] 多维度对比矩阵
                  │
                  ▼
              [关键洞察] 提炼竞争规律/壁垒/机会
                  │
                  ├─ deep/strategic ──► [趋势预测] 功能/用户/商业化趋势
                  │                        │
                  └─ basic/standard ──┐    │
                                     ▼    ▼
                                 [战略报告] 编译最终报告
                                     │
                                     ▼
                                    END
```

## 数据源

- **搜索引擎**: DuckDuckGo（中英文双语，无需 API key）
- **搜索策略**: 每节点 3-6 次搜索，深度模式增加搜索量
- **总搜索量**: basic ~20次, standard ~25次, deep ~35次, strategic ~45次

依赖: `pip3 install ddgs langgraph`

## 注意事项

- 搜索数据来自公开网络，可能存在时效性和准确性限制
- 洞察和趋势判断基于 Claude 推理，非确定性结论
- 若 DuckDuckGo 搜索受限，部分节点可能数据不足，报告会标注
- 结论仅供研究参考，重大商业决策请结合更多信源验证
